Floating Point System (\(F\))

A computer does not keep real numbers in their full mathematical form. Instead, it records a structured approximation made of three parts: a sign, a finite sequence of digits, and an exponent.

The general form (base \(\beta\) )

Using base \(\beta\), any non-zero number \(x\) in a floating-point system is written as

\[ x = s \cdot (.a_1a_2\ldots a_t )_\beta \cdot \beta^{\,e} \] \[ x = (\text{sign})\cdot(\text{fractional part})\cdot(\text{exponent}) \]

where the pieces follow these rules:

  • \(s = -1\) or \(s = +1\)
  • \(0 \le a_i \le \beta - 1\)
  • \(L \le e \le U\), with \(e\) an integer and typically \(L<0<U\)

What does the “fractional part” mean?

The string \((.a_1a_2\ldots a_t)_\beta\) is just shorthand for a finite base-\(\beta\) expansion:

\[ (.a_1a_2\ldots a_t)_\beta = \frac{a_1}{\beta} + \frac{a_2}{\beta^2} + \cdots + \frac{a_t}{\beta^t} \]

Normalization

We usually assume \(a_1 \ne 0\). That condition is called a normalized floating-point system. The normalization condition ensures uniqueness of representation by preventing more than one digit sequence from corresponding to the same number, such as those obtained by introducing leading zeros in the fractional part.

How we name a floating-point system

The parameters \((\beta, t, L, U)\) specify the arithmetic characteristics of the floating-point system:

  • \(\beta\): base
  • \(t\): number of digits kept in the fractional part (precision)
  • \(L, U\): smallest and largest allowed exponents (range)

In such a system, a floating-point number is determined by the tuple \(\{s, a_1, a_2, \ldots, a_t, e\}\).

On modern computers, the base \(\beta = 2\), i.e., we use the binary number system. There are only two digits, \({0, 1}\), which can be easily represented in computer hardware

Two unavoidable consequences

  1. Not every real number can be represented.
    • So there are gaps in \(F\).
  2. Very small and very large numbers can't be represented.
    • Because the exponent is trapped between \(L\) and \(U\).

In spite of these limitations, modern computers have enough precision in representing real numbers which allows us to obtain useful numerical answers to mathematical equations modeling nature.
